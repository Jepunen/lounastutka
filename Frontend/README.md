# Frontend Documentation Landing Page

This folder contains the Lounastutka frontend: a Vite + React + TanStack Router single-page app focused on browsing lunch places on a map and list view.

Use this file as the starting point for frontend development, architecture, and deployment notes.

## What This Frontend Does

- Shows lunch places on an interactive Leaflet map.
- Provides a searchable and filterable list view.
- Supports user location-based distance sorting.
- Includes authentication UI for password and passkey (WebAuthn) flows.
- Uses a mobile-friendly bottom navigation between app sections.

## Tech Stack

- React + TypeScript
- Vite
- TanStack Router (file-based routing)
- Tailwind CSS
- Leaflet + React Leaflet
- Framer Motion
- WebAuthn browser support via `@simplewebauthn/browser`

## Project Structure

```text
Frontend/
	app/
		components/        # Shared UI and map components
		data/              # Local demo/test restaurant data
		hooks/             # Custom hooks (e.g. auth)
		routes/            # Route components (/, /list, /settings, /components)
		services/          # API and auth service helpers
		utils/             # Utility functions (distance, etc.)
		app.css            # Global styles
		main.tsx           # App bootstrap + router provider
		routeTree.gen.ts   # Auto-generated by TanStack Router
	docker/nginx/        # Nginx config for container runtime
	public/              # Static assets
	Dockerfile           # Frontend production image
```

## Routes

- `/` - Map-first home view
- `/list` - Searchable/filterable restaurant list
- `/settings` - Authentication/settings view
- `/components` - Component playground/demo page

## API Integration Notes

- Frontend API helpers call same-origin routes under `/api`.
- JWT is stored in `localStorage` under key `jwt`.
- Authenticated requests include `Authorization: Bearer <token>` when available.
- A `401` response clears the stored token automatically.

This means local development should run frontend and backend together (for example via repository-level Docker Compose), or provide equivalent proxying/routing for `/api`.

## Local Development

From this directory (`Frontend/`):

```bash
npm install
npm run dev
```

Default dev URL:

- `http://localhost:5173`

Available scripts:

- `npm run dev` - Start Vite dev server
- `npm run build` - Create production build in `build/client`
- `npm run typecheck` - Run TypeScript checks

## Recommended Full-Stack Local Run

From repository root, start all services together:

```bash
docker compose -f compose.dev.yaml up
```

This is the easiest way to get frontend + backend + database running with expected routing.

## Production Build Output

- Vite output directory: `build/client`
- Served by Nginx in containerized deployments

## Docker Image (Frontend only)

Build from repository root:

```bash
docker build -t lounastutka-frontend:latest ./Frontend
```

The production image:

- Builds static assets with Node
- Serves assets with Nginx
- Uses SPA fallback (`try_files ... /index.html`)
- Proxies `/api` and `/auth` to backend service inside container network

## Notes for Contributors

- Do not manually edit `app/routeTree.gen.ts` (generated file).
- Keep route files inside `app/routes` so TanStack Router codegen stays in sync.
- Prefer placing API logic in `app/services` and UI logic in components/hooks.
